Binary Tree Maximum Path Sum

2026-07-08 ps트리dfs

서론

| https://neetcode.io/problems/binary-tree-maximum-path-sum/question

가장 합이 큰 path를 찾으시오

푸는 방법

처음에는 2차원 dp 테이블을 만들어 root를 거치는 것과 안 거치는 것을 구별하면서 가야한다고 생각했었다.

하지만 그럴 이유가 없는 것이 단순히 이어지는 경로와 끊기는 경로만 생각하면 되기 때문이다.

이어지는 경로

root로 이어진다는 것은 left -> root -> root의 부모 노드 ... 이런 식으로 되는 것이다.

그렇다면, 이 값은 계속 이어져야 하니 dfs의 return 값으로 사용하면 된다.

끊기는 경로

끊긴 다는 것은 left -> root -> right ... 이런 식으로 되는 것이다. 우리는 dfs 특성 상 left와 right는 거의 동시에 탐색하므로 left + root + right를 하면 된다.

이 두 가지를 적절하게 dfs로 구현하면 된다.

다음은 코드이다.

class Solution {
    int max_sum = -1e9;

    int dfs (TreeNode* node) {
        if (node == nullptr) return 0;

        int left = max(0, dfs(node->left));
        int right = max(0, dfs(node->right));

        max_sum = max(max_sum, left + right + node->val);

        return max(left, right) + node->val;
    }
public:
    int maxPathSum(TreeNode* root) {
        dfs(root);
        return max_sum;
    }
};

\(max\_sum\) 이란 변수를 두어 끊기는 경로를 기록해 나간다. return은 계속 이어지는 경로를 반환한다.

dfs(root)에서 max_sum에 끊기는 경로가 들어갈 것이므로 본 함수에서 max_sum을 반환해주면 된다.

결론

무작정 dp 테이블을 세우려고 했는데, 잘못된 판단이었다. 경로에 대한 문제는 경로를 어떻게 전달할 지를 더 고민하고 풀어야겠다.
