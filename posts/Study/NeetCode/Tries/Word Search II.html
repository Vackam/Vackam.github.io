Word Search II

2026-07-12 pstrie트리

서론

| https://neetcode.io/problems/search-for-word-ii/question

단어 목록 중 2차원 배열을 순회하며 만들 수 있는 단어를 찾아라

푸는 방법

Trie를 무얼로 구성할 지가 중요한 문제이다.

주어진 words를 Trie로 구성한 다음에 board의 각 모든 점에서 출발하는 dfs를 구성하면 풀 수 있다.

다음은 코드이다.

struct Node {
    Node* children[26] = {};
    string word;
};
class Solution {
private:
    vector<int> dir_x = {1, -1, 0, 0};
    vector<int> dir_y = {0, 0, 1, -1};
public:
    void dfs(int x, int y, Node* node, const int& n, const int& m, vector<vector<char>>& board, vector<string>& result) {
        // node는 현재 위치이고, board[x][y]를 탐색해야 한다.
        Node* a = node->children[board[x][y] - 'a'];
        if (!a) return;
        if (!a->word.empty()) {
            result.push_back(a->word);
            a->word.clear();
        }

        char temp = board[x][y];
        board[x][y] = '#';
        for (int i=0; i<4; ++i) {
            int nx = x + dir_x[i];
            int ny = y + dir_y[i];

            if (nx < 0 || nx >= n || ny < 0 || ny >= m
                || board[nx][ny] == '#') continue;
            dfs(nx, ny, a, n, m, board, result);
        }

        board[x][y] = temp;
    }
    vector<string> findWords(vector<vector<char>>& board, vector<string>& words) {
        Node* root = new Node();        

        for (auto& w : words) {
            Node* cur = root;
            for (auto c : w) {
                if (!cur->children[c - 'a']) {
                    cur->children[c - 'a'] = new Node();
                }
                cur = cur->children[c - 'a'];
            }
            cur->word = w;
        }

        int n = board.size();
        int m = board[0].size();
        vector<string> result;

        for (int i=0; i<n; ++i)
            for (int j=0; j<m; ++j) 
                dfs(i, j, root, n, m, board, result);

        return result;
    }
};

4방향 탐색을 위해 변수를 선언해 주었고, dfs를 통해 해결책을 찾았다. 또한 board 자체를 visited 배열로 사용하였다.

결론

사실 처음에 삽질을 많이 했다.

bfs로 board에서 순회하면서 words를 찾는 것이 정답이라 생각해서 그렇게 했는데, 문제는 visited 였다. visted를 최대 3만번을 복사해서 가지고 다녀야 하니 답이 없었다.

그래서 여기서 visited를 최소한으로 유지할 수 있는 dfs + 백트래킹을 생각했어야 했다.

또한 board를 통해 Trie를 구성하거나 탐색한다고 가정한다면 마찬가지로 답이 없다.

Trie를 구성한다고 하면 가능한 모든 path를 구해서 저장해야 하고, Trie를 탐색할 때 path 당 string을 기억하면서 가야한다. 이런 문제점에서 words를 Trie로 만들어야 한다는 것을 깨달아야 했다.

이걸 못해서 삽질을 많이했다... 안되는 걸 붙잡지 말고 "아 이거 안되네? 이거 해봐야 겠다." 이런 식으로 접근해야겠다.
