Combination Sum

2026-07-13 ps백트래킹

서론

| https://neetcode.io/problems/combination-target-sum/question

중복 조합으로 target을 만드는 경우를 찾아라.

푸는 방법

백트래킹으로 작성하면 풀린다.

나는 하나의 index에서 넣을 수 있는 모든 가능성을 시도해보는 식으로 접근했다.

예를 들어 처음이 2라는 값을 가지고 목표 값이 9라고 했을 때,

0 -> dfs

2 -> dfs

4 -> dfs

6 -> dfs

8 -> dfs

로 풀었다.

다음은 코드이다

class Solution {
public:
    vector<vector<int>> combinationSum(vector<int>& nums, int target) {
        int size = nums.size();
        vector<vector<int>> result;
        vector<int> values;

        auto dfs = [&] (auto& self, int now_val, int now_index) -> void {
            // now_val은 합산해서 온 값. 값이 넘으면 제껴야 할 듯
            // 같아도 넣는 작업은 초반에 하자.
            if (now_val == target) {
                result.emplace_back(values);
                return;
            }
            if (now_val > target) return;
            if (now_index == size) return;

            int sum_val = nums[now_index];
            int counter = 0;

            while (now_val <= target) {
                // 타겟과 같을 때까지만 자신을 최대한 넣어본다.
                self(self, now_val, now_index+1);

                now_val += sum_val;
                values.emplace_back(sum_val);
                counter++;
            }

            values.resize(values.size() - counter);
        };

        dfs(dfs, 0, 0);

        return result;
    }
};

이것보다 더욱 단순하게 풀 수 있는데 자신을 포함할 것인지 아닌지만 정하면 되는 코드가 있다.

즉, 인덱스는 순서대로 가되 이것을 사용 여부를 판단하여 푸는 것이다.

또한, 중복해서 사용할 수 있으므로 인덱스를 업데이트 안하는 것이 핵심이다.

다음은 해당 내용의 dfs 람다 코드이다.

auto dfs = [&](auto& self, int idx, int remain) -> void {
	if (remain == 0) { result.push_back(cur); return; }
	if (remain < 0 || idx == n) return;
	// nums[idx]를 쓴다 (재사용 가능하니 idx 그대로)
	cur.push_back(nums[idx]);
	self(self, idx, remain - nums[idx]);
	cur.pop_back();
	
	// nums[idx]를 안 쓴다 (다음으로)
	self(self, idx + 1, remain);
};
