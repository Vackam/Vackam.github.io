Generate Parentheses

서론

| https://neetcode.io/problems/generate-parentheses/question?list=neetcode150

n개의 유효한 소괄호 조합을 찾아라.

푸는 방법

괄호하면 바로 스택이 생각나게 된다.

스택의 맨 위가 열린 괄호라면 열거나 닫는 동작이 가능하고, 아무것도 없을 시에는 여는 동작만 가능하다. 또한 닫았을 때는 여는 괄호 하나를 pop해준다.

위 설명이 괄호 문제의 정석같은 접근이다.

하지만, 이 문제에서는 가능한 괄호 조합을 직접 구해서 출력해야 한다. 그러므로 pop 연산을 할 수 없다.

그렇기에 열려있는 괄호의 개수와 지금까지 총 괄호의 개수를 트래킹하여 문제를 풀어야 했다.

다음은 코드이다.

class Solution {
    vector<string> result;
public:
    void dfs (string now, int counter, int open, int n) {
        if (open == 0 && counter == n) {
            result.push_back(now);
            return;
        }

        if (open >= 1) {
            now.push_back(')');
            dfs(now, counter, open - 1, n);
            now.pop_back();
        }
        if (counter < n) {
            now.push_back('(');
            dfs(now, counter + 1, open + 1, n);
            now.pop_back();
        }
        return;
    }
    vector<string> generateParenthesis(int n) {
        string start = "(";
        int counter = 1;
        int open = 1;

        if (n == 1) {
            result.push_back("()");
            return result;
        }
        dfs(start, counter, open, n);
        return result;
    }
};

전에 다른 문제와 비슷하게 path를 now로 해서 트래킹 해주었다. 여는 경우 닫는 경우를 모두 체크했다.

답지를 보니 dp로 푸는 방법도 있었다.

0~n개로 가능한 경우가 전부 저장되어 있을 때 n+1은 다음과 같이 기술할 수 있다.

\(dp[n+1]= \sum_{i=0}^{n} ( + dp[i] +) + dp[n-i]\)

보기는 안좋지만 이렇게 풀 수 있다.
