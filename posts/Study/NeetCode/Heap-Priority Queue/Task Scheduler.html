Task Scheduler

2026-07-21 ps

서론

| https://neetcode.io/problems/task-scheduling/question

cpu 스케쥴러를 작성하라. 대신 같은 작업을 바로 반복할 수 없다.

푸는 방법

생각해보면 가장 많이 나오는 친구를 기준으로 판별하면 된다.

문제 예제 중 AAABBC에 k=3인 경우를 생각해보자.

A _ _ | A _ _ | A

이 구조는 우선 확정이다. 이후 B와 C를 걸리지 않게만 넣어버리면 된다.

공식화 하면

블록 하나당 크기가 n+1, 블록이 총 mx -1, 만약 A와 빈도가 동일한 놈이 있으면 뒤에 몇 번 더 붙여야 하므로 count.

최빈값이 아닌 애들은 _ 자리에 그냥 넣으면 된다.

그리고 위에서 구한 값과 tasks의 사이즈를 분석해야 하는데, 사이즈가 그냥 크다면 이렇게 복잡하게 알 것 없이 그냥 집어 넣으면 되기 때문이다.

다음은 코드이다.

class Solution {
public:
    int leastInterval(vector<char>& tasks, int n) {
        vector<int> cnt(26, 0);
        for (char c : tasks) cnt[c - 'A']++;
        int mx = *max_element(cnt.begin(), cnt.end());
        int mx_count = count(cnt.begin(), cnt.end(), mx);
        return max((int) tasks.size(), (mx - 1) * (n + 1) + mx_count);
    }
};

결론

처음에는 하나하나 트래킹하면서 세야하는 줄 알았는데, 칸막이 이론 들고 와서 빠르게 처리할 수 있었던 문제이다.
