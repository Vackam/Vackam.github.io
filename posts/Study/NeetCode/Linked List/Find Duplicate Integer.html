Find Duplicate Integer

2026-07-01 ps

서론

| https://neetcode.io/problems/find-duplicate-integer/solution

배열에 단 하나의 수만이 중복되어 있는 데 그것을 찾으라.

푸는 방법

전에 계속 하던 것처럼 해시로 해도 되고, 무작정 sorting 후 생각해도 된다.

근데 문제에서 space complecity를 \(O(1)\)로 하라고 해서 bit-wise 연산을 생각했다.

아래는 내 처음 코드인데, 문제점이 있지만 통과는 된다.

int findDuplicate(vector<int>& nums) {
	unsigned int bit_think = 0;

	for (auto& a : nums) {
		unsigned int indicator = 1 << (a - 1); // Problem 2
		if ((bit_think & indicator) > 0) return a; // Problem 1
		bit_think = bit_think ^ indicator;
	}
	return 0;
}

두 가지 문제점이 있는데, 하나하나 집어보겠다.

Problem 1 연산자 우선순위

처음에는 코드를 아래와 같이 했는데, 항상 두 번째 값을 반환하는 버그가 있었다.

if (bit_think & indicator > 0) return a;

문제는 연산자 우선순위에 있었다. 대소 비교가 비트 계산보다 빨라서 컴파일러가 다음과 같이 해석했다.

if (bit_think & (indicator > 0)) return a;

그래서 실패한 것이다.

해결 방법은 위에서 제시한 제출 코드와 같이 괄호로 묶어주면 된다.

Problem 2 bit-wise의 한계

아마 테스트 케이스가 적어서 그런 거 같은데, 위 제출 코드는 정답이 33 이상일 때 걸러내질 못한다. 대부분 8바이트 일 것이기 때문이다..

그래서 아래와 같이 푸는 것이 상식적인 풀이일 듯 하다.

vector<bool> (nums.size(), false);

// num 마다 체크 하면서 가기..

솔루션에서는 토끼와 거북이 방식을 추천하고 있다.

토끼가 두 칸씩 가고, 거북이가 한 칸씩 진행한다.

배열을 링크드리스트라 생각하면 loop가 생겨 결국 어디선가 만나게 된다.

이후 토끼를 시작 지점으로 보내고 다시 만날 때까지 진행한다.

그러면 중복된 숫자를 가리키게 된다.
